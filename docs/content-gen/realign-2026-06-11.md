## Verified high-impact gaps (all claims checked against code)

1. **music-bed-broken** [small] — `getSfx('bed', totalDur)` has no max clamp →
   ElevenLabs 400 → silent video. Fix: clamp ≤22s + loop via `-stream_loop`
   (port the looped-bed + alimiter mixer that ALREADY EXISTS in
   `audio-bed.ts:71-204`).
2. **ducking-missing** [medium] — static amix weights (`volume=0.25`, "4 1").
   Fix: `sidechaincompress` keyed by the concat voice track (−6dB, release 200ms)
   in the stage-3 filtergraph; per-slot `duck_depth` flag.
3. **sfx/music registry holes** [small] — only whoosh/ding/soft_chimes resolve;
   MUSIC_TOKENS have ZERO overlap with sfx.ts registry; `ascending_electronic_sting`
   unresolvable (nearest: `ascending_sting`). Fix: add generator entries, export
   ONE canonical enum consumed by tools.ts + sfx.ts, build-time assertion.
4. **music-multitrack-rotation** [large] — spec: track switches at every niche
   boundary, mood-matched. Fix: music segment list in video_compose args
   (`{start_slot,end_slot,token}`), builder emits per-niche moods, afade 0.5s at
   boundaries. Stage: start with body+CTA two-mood version.
5. **recipe-demo / mini_player b-roll** [large] — 21% of the reference; we ship 0.
   CORRECTED by verifier: `recipe-showcase.ts` already generates transcript-grounded
   `{narration, clip_start, clip_end}` beats (cached in `content_gen_recipe_showcase`),
   and `timeline.ts:236-259` already compiles mini_player segments with
   `diegetic:true` + `duck_under_diegetic` — but only in the JSON timeline path,
   never wired into the producer. Missing pieces: (a) wire
   `getOrGenerateRecipeShowcase` into `buildListicleScript`, (b) `clip_extract`
   tool (yt-dlp + ffmpeg trim — download infra exists), (c) mini-player frame as
   ffmpeg overlay in video-compose (NOT a static card), (d) remove `-an` at
   video-compose.ts:~523 and mix source audio at −15dB under VO.
6. **proof-beats-static-stretch** [medium] — both proofs forced onto the same
   about_page on white; 10s without a cut. Fix: proof_1 = channel_page on
   dark_gray + subscriber annotation; proof_2 = about panel; MG-style sequential
   3-stat highlight chain (~1.25s/stat — pixel-scan already locates all 3 rows).
7. **pacing-hold-budget** [medium] — holds derive solely from narration spans.
   Fix: hold-budget pass after slicing: clamp non-exempt slots to 2.5s, split
   long narration spans across extra visuals (b-roll, second highlight, rapid card).
8. **narration-fixed-templates** [large] — builder hand-authors fixed strings;
   writer only gets 3 stub beats. Fix: writer authors all beats per skeleton
   schema + 9 phrase banks + rolling 50-phrase anti-repetition window (DB table).
9. **dollar-trio-data-slots** [medium] — only lump_sum fires; spec requires ≥1 of
   yearly/daily/monthly + upload_rate + age + growth. Fix: derive in builder data
   prep from existing DB columns; feed writer variable bundle.
10. **channel-b reprise** [large] — spec default 2 channels/niche; we do 1:1.
    Fix: group input channels by cluster, compressed channel-B block + optional
    saturation montage. (Defer until single-channel niches are dense.)
11. **cta-structure** [medium] — no subscribe slot; `neutral` ignores bg →
    #111-on-#282828 invisible text; "the 1 faceless" digit bug; wrong icon/bg
    on pitch card.
