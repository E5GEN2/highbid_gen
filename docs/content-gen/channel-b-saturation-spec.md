Optional clauses (appended, each ≤1/8–3/8):

### 3C. Text-card ↔ narration sync (both beats)

## 5. PACKAGING RULES (invariants)

1. **YouTube DARK theme, 14/14.** Every YT surface (#0F0F0F page, #0D0D0D cards, gray #AAA meta text) — never light theme, regardless of canvas.
2. **Card-on-canvas framing, 14/14.** All YT UI, footage, memes, infographics render as rounded-corner cards (radius ~8–16px @640w ≈ 24–48px @1080p) floating centered on a flat full-bleed canvas. UI is never full-bleed. No drop shadows (or sub-resolution subtle). One exception: transparent-bg vector assets (SC5 calendar, SC2 emoji crowd, flork doodle) composite directly on canvas without a card.
3. **Exactly two canvas colors:** near-white **#FDFDFD** (6/14 instances) and flat warm dark gray **#3B3B3B–#3E3E3E** (7/14; SC2 uses both). Canvas is constant within a beat except meme punchlines may flip it (CB8).
4. **Text cards are full-bleed canvas-colored:** dark canvas → white bold text; white canvas → black bold text. Dark card bg is the EXACT canvas gray, so screenshots read as cards and text reads as the continuous canvas speaking (explicit in SC1, SC3, SC5, SC6). Font: heavy geometric sans (SF Pro/Helvetica-Bold class), single line, horizontally centered, baseline at ~48–55% frame height, cap height ~5.5–10% of frame height, lowercase except sentence-initial caps; italic only for emphasis builds (1/14).
5. **Screenshot hygiene:** no browser chrome, no YouTube sidebar/top bar, no scrollbar, no cursor (except SC5 analytics screen-capture), content-column-only crops; pages may clip mid-grid/mid-item at card edges. Grids are desktop 4-column, 2–5 rows.
6. **Card geometry norms (640w):** full channel page ~56–69% frame width x 76–86% height; header chip ~48–49% width, ~3:1; zoom/grid-only cards wider at 73–82% width; footage/meme cards 38–57% width (16:9, 4:3, or near-square); single-video-card cutout ~34% width keeping its own #0F0F0F backing.
7. **Watermarks:** none added by editor, 14/14. Channel-owned watermarks inside source thumbnails are preserved (they double as branding-consistency proof).
8. **Edit grammar:** hard cuts only; static holds; cuts every 0.45–3.1s (text cards 0.45–1.65s, pages 1.3–3.1s, montage pages ~0.6s); scrolls constant-velocity inside fixed viewports; no ken-burns (1 exception, SC5); no drawn annotations ever.
9. **Mock pages allowed:** screenshots may be recreated rather than captured (CB2 explicitly mock: flat-color gag banner, monogram avatar) but must be pixel-faithful to YT dark desktop layout including stats-line format `@handle · N subscribers · N videos`, tab underlines, chip states, kebab menus, duration badges.
10. **Generator data contract per channel render:** name, handle, sub count, video count, bio line, avatar, banner image (optional; subscribe-gag variant allowed), button set (Subscribe [+Join] [+Visit community]), tab-row variant (full Home/Videos/Shorts/Playlists/Posts vs Videos-only), sort-chip state, and 6–20 videos each with {title (uniform template), view count, relative age, duration, thumbnail (uniform template, optional channel watermark)}. Optional: about-panel block {country, join date, total views, Share button}. Saturation additionally needs: 1–3 sibling-channel datasets, or 3–4 search-result items {thumbnail, full title, views, age, channel name+avatar, description snippet, optional Subtitles/4K/chapters badges}, contrast-channel grid (8 videos, weak views), and the infographic/meme asset library (LOW/HIGH gauge, calendar flip, YT Studio views graph with tooltip series, person-at-laptop crowd grid, effort-typing memes, chess footage).
11. **Numbers live on screen, claims live in voice** — the single deepest invariant: across all 14 instances, 0 subscriber counts and 0 exact view counts are ever spoken; narration supplies the frame ("another channel", "same format", "performing well", "competition is low") and the rendered UI supplies every digit, with cut timing (or a 0.6–1.05s silent dwell) pointing the eye at the number at the moment the claim lands.
